# Human Design Natal Bot

Каркас Telegram-бота как мини-сервиса: бот собирает данные рождения, отдельные сервисы получают координаты и timezone, расчет карты сохраняется JSON-ом, интерпретация генерируется отдельно, история хранится в локальном JSON-файле.

## Что уже сделано

- `bot/` - диалог Telegram: дата, время, место, история запросов.
- `services/geoService.js` - поиск места через Nominatim/OpenStreetMap.
- `services/chartService.js` - отдельный слой расчета карты.
- `services/interpretationService.js` - отдельный слой текста трактовки.
- `db/database.js` - локальная JSON-база: пользователи и история расчетов.
- `.env.example` - шаблон настроек.

## Как запустить

1. Установи Node.js 20+.
2. В папке проекта выполни:

```bash
npm install
```

3. Создай файл `.env` рядом с `.env.example`:

```env
BOT_TOKEN=токен_от_BotFather
DATABASE_PATH=./data/bot.json
NOMINATIM_EMAIL=твоя_почта@example.com
NOMINATIM_USER_AGENT=HumanDesignNatalBot/0.1
EPHEMERIS_PATH=./ephe
FULL_REPORT_STARS=150
ADMIN_TELEGRAM_IDS=123456789
```

## Аналитика контента

Для каждой публикации используйте отдельную deep-link ссылку, например:

```text
https://t.me/ИМЯ_БОТА?start=ig_rel_001
```

Бот сохраняет первый и последний источник пользователя и события воронки: запуск, начало и завершение расчёта, показ оффера, начало и подтверждение оплаты, выдачу отчёта. Команда `/analytics` показывает уникальных пользователей по этапам и выручку за последние 7 дней. Она доступна только Telegram ID из `ADMIN_TELEGRAM_IDS` (несколько ID указываются через запятую).

### Автоматическая запись в Google Sheets

1. В нужной Google-таблице откройте `Расширения` → `Apps Script`.
2. Вставьте содержимое `integrations/google-apps-script/Code.gs` вместо стандартного кода.
3. В настройках проекта Apps Script добавьте свойство скрипта `ANALYTICS_SECRET` с длинным случайным значением.
4. Нажмите `Развернуть` → `Новое развертывание` → `Веб-приложение`. Выполнение: от вашего имени; доступ: все пользователи.
5. Скопируйте URL развертывания, который заканчивается на `/exec`.
6. Добавьте в Railway:

```env
GOOGLE_SHEETS_WEBHOOK_URL=https://script.google.com/macros/s/.../exec
GOOGLE_SHEETS_SECRET=то_же_значение_что_в_ANALYTICS_SECRET
```

После первого события скрипт автоматически создаст листы `Статистика` и `События`. Ошибка или недоступность Google Sheets не останавливает работу бота: событие всё равно сохраняется в локальной базе.

4. Запусти:

```bash
npm start
```

## Как подключить к Railway

1. Создай новый проект на Railway.
2. Подключи репозиторий GitHub с этим ботом или загрузи проект через Railway CLI.
3. В Railway открой `Variables` и добавь:

```env
BOT_TOKEN=токен_от_BotFather
DATABASE_PATH=./data/bot.json
NOMINATIM_EMAIL=твоя_почта@example.com
NOMINATIM_USER_AGENT=HumanDesignNatalBot/0.1
EPHEMERIS_PATH=./ephe
```

4. Railway сам возьмет команду запуска из `railway.json`: `npm start`.
5. После деплоя открой логи сервиса. Если все хорошо, там будет строка `Human Design Natal bot is running.`

Важно: текущая история расчетов хранится в локальном JSON-файле. На Railway такое хранение подходит только для теста, потому что файлы могут потеряться при redeploy. Для нормального продукта лучше подключить PostgreSQL в Railway и перенести историю туда.

## Что требуется от тебя

1. Получить Telegram bot token у BotFather и вставить его в `.env`.
2. Указать email в `NOMINATIM_EMAIL`. Это хорошая практика для Nominatim, чтобы сервис мог связаться с владельцем приложения при проблемах.
3. Решить, нужен ли сразу настоящий астрологический расчет или пока достаточно проверить сценарий бота.

## Важный момент про расчеты

Сейчас `chartService.js` работает в двух режимах:

- Если `swisseph` установлен и настроен, сервис пробует считать через Swiss Ephemeris.
- Если `swisseph` не установлен или не завелся, бот возвращает демо-карту с явной пометкой `Это демо-расчет`.

Для настоящего продукта нужно отдельно довести Swiss Ephemeris:

1. Проверить лицензию Swiss Ephemeris под твою коммерческую или некоммерческую модель.
2. Скачать ephemeris-файлы в папку `ephe/`.
3. Проверить точность UTC/timezone-конвертации для даты рождения.
4. Сравнить несколько карт с Astro.com или другим эталоном.
